# Color Guidelines

## Primary Palette

memoria.uy uses a **strictly monochrome** palette for maximum clarity and timelessness.

### Black
- **Hex**: `#000000`
- **RGB**: `0, 0, 0`
- **Usage**: Logo foreground, borders, text

### White
- **Hex**: `#FFFFFF`
- **RGB**: `255, 255, 255`
- **Usage**: Logo background, page background

## Logo Variants

### Black on White (Primary)
Use this for:
- Light backgrounds
- Print materials
- Most digital contexts

### White on Black (Inverse)
Use this for:
- Dark backgrounds
- Dark mode interfaces
- High contrast scenarios

## Important Rules

- **Never use colors**: The logo must remain black and white
- **Never use gradients**: Only solid fills
- **Never add transparency**: Use the appropriate variant (black-on-white or white-on-black) instead
- **Minimum contrast**: Always ensure sufficient contrast with background (WCAG AA minimum)

## Background Colors (Web Interface)

While the logo is strictly B&W, the web interface uses minimal accent colors:

### Gray Scale
- **Gray 50**: `#FAFAFA` - Subtle backgrounds
- **Gray 100**: `#F5F5F5` - Card backgrounds
- **Gray 400**: `#BDBDBD` - Secondary text (e.g., "// comments")
- **Gray 600**: `#757575` - Muted text
- **Gray 700**: `#616161` - Body text
- **Black**: `#000000` - Headers, borders

These grays are for UI only, never for the logo itself.

## Vote Button Colors (Optional)

Vote buttons may use minimal color for UX clarity:
- **Good**: Green accent (optional, can remain monochrome)
- **Bad**: Red accent (optional, can remain monochrome)
- **Neutral**: Gray (default)

These are UI elements, not part of the brand identity.
